# Slide Whitepaper

サービス紹介ホワイトペーパー（12-15枚）の作成ワークフロー。

**参照スキル**:
- timewell-design-system（デザイン基盤）
- nanobanana-core（技術基盤）

---

## ワークフロー（7ステップ）

### Step 0: 日付確認と最新情報チェック

**必ず最初に実行する。**

```markdown
1. Web検索で今日の日付を確認
   - 検索: "今日の日付"
   - 目的: 情報の鮮度を保証

2. 最新情報の確認
   - サービス公式ページの更新日
   - PR TIMESの最新プレスリリース
   - 数値データの有効期限

3. 確認結果を記録
   - 確認日: YYYY-MM-DD
   - 情報の鮮度: ◯最新 / △要確認 / ✕古い
```

### Step 5: デザインガイドライン確認

`timewell-design-system` を参照し、以下を確認:
- ブランドカラー
- グラスモーフィズム仕様
- Bento UIレイアウト
- 禁止事項

### Step 6: Nano Banana Pro生成

`nanobanana-core` を参照し、スライド画像を生成。
- 待機時間: 40秒
- レート制限対策を適用

### Step 7: PPTX組み立て

`nanobanana-core` の共通処理を使用。

## 標準スライド構成（12枚）

| # | タイトル | 内容 | レイアウト |
|---|---------|------|-----------|
| 1 | 表紙 | サービス名、キャッチコピー | 中央配置 |
| 2 | 課題 | 想定顧客の課題4つ | Bento UI 2x2 |
| 3 | ソリューション | サービス概要+スクショ | 左テキスト+右画像 |
| 4 | 技術 | 仕組み・技術説明 | Bento UI |
| 5 | 機能① | 主要機能1 | 左テキスト+右数値 |
| 6 | 機能② | 主要機能2 | 左テキスト+右数値 |
| 7 | 機能③ | 主要機能3 | 左テキスト+右数値 |
| 8 | 導入効果 | Before/After表 | テーブル |
| 9 | 導入事例 | 2社の事例 | Bento UI 1x2 |
| 10 | セキュリティ | 安心感要素 | Bento UI 2x2 |
| 11 | 料金プラン | 2プラン比較 | Bento UI 1x2 |
| 12 | CTA | 問い合わせ導線 | 中央カード |

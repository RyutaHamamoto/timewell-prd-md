# Slide Research

リサーチレポートスライド（100枚以上）の作成ワークフロー。

**参照スキル**:
- timewell-design-system（デザイン基盤）
- nanobanana-core（技術基盤）

---

## 用途

- 市場調査レポート
- 競合分析
- 技術トレンド調査
- 業界動向まとめ
- ウェブリサーチの集約

**特徴**: 大量の情報を構造化、出典管理、データビジュアライゼーション

---

## ワークフロー（9ステップ）

### Step 0: 日付確認と調査期間設定

```markdown
1. Web検索で今日の日付を確認
2. 調査期間の設定
   - 調査開始日:
   - 調査終了日:
```

### Step 6: 分割生成戦略

100枚以上のスライドを効率的に生成する。

```javascript
// バッチ処理
const BATCH_SIZE = 10;  // 10枚ずつ生成
const BATCH_WAIT = 300000;  // バッチ間5分待機

async function generateInBatches(slides) {
  const batches = [];
  for (let i = 0; i < slides.length; i += BATCH_SIZE) {
    batches.push(slides.slice(i, i + BATCH_SIZE));
  }

  for (let i = 0; i < batches.length; i++) {
    console.log(`Batch ${i + 1}/${batches.length}`);
    for (const slide of batches[i]) {
      await generateSlide(slide.prompt, slide.path);
      await wait(40000);  // 40秒待機
    }
    if (i < batches.length - 1) {
      console.log('Batch complete. Waiting 5 minutes...');
      await wait(BATCH_WAIT);
    }
  }
}
```

### Step 7: データビジュアライゼーション

リサーチレポートでは、データの視覚化が重要。

```markdown
## グラフの種類と用途

| データタイプ | 推奨グラフ |
|------------|-----------|
| 時系列推移 | 折れ線グラフ |
| 構成比 | 円グラフ、棒グラフ |
| 比較 | 棒グラフ |
| 相関 | 散布図 |
| 分布 | ヒストグラム |
| プロセス | フローチャート |
| 関係性 | マトリクス、ベン図 |
```

### Step 8: 出典管理

```markdown
## 出典記載ルール

### スライド内
- 右下に小さく出典を記載
- 形式: "出典: {ソース名} ({年})"

### 参考文献スライド
- 全ての出典をリスト化
- URL、取得日を明記

### 信頼度の表示
- ★★★: 公式発表、IR資料
- ★★☆: 大手メディア、調査会社
- ★☆☆: ブログ、SNS
```

### Step 9: レビューと品質チェック

- データの正確性確認
- 出典の再確認
- ストーリーの一貫性
- 重複の排除
